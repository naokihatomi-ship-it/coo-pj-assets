# coo-pj-assets

社内スライドジェネレータが参照するブランド画像アセットを集約したリポジトリ。

## 構成

```
brands/
├── atomic/         Atomic Software（コーポレート）
├── keibiforce/     KeibiForce（プロダクト）
└── medicalforce/   MedicalForce（プロダクト）
```

## 命名規則

| サフィックス | 用途 |
|---|---|
| `logo` | 通常背景（明るい背景）用フルロゴ |
| `logo-white` | 暗い背景用の白抜きフルロゴ |
| `mark` | ロゴタイプを除いたシンボル単体（medicalforce のみ） |
| `watermark-*` | スライド地紋用。**α を焼き込み済み**（半透明） |

## 重要な注意

- `watermark-*.png` はすでに半透明アルファをベイク済み（commit `addc621`）。利用側で重ねて透過させると二重に薄くなる。
- 各 PNG は切り抜き調整済み（commit `8b468f7`）。再クロップ前に元データの所在を確認すること。
- ファイルはルート直下フラットから `brands/<brand>/` 配下へ再編済み。古いパスを参照するスライドジェネレータ側コードは追従が必要。

## 追加・変更時のルール

1. ブランドが増える場合は `brands/<new-brand>/` を作って同じ命名規則で配置する。
2. 既存ファイル名は変えない（外部参照を壊さないため）。差し替えは同名 PNG の上書きで行う。
3. 透過処理を伴う変更（watermark 等）は、ベイク済みかどうかをコミットメッセージに明記する。
